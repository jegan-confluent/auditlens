# Host-venv requirements for the AuditLens setup wizard.
#
# The wizard (./setup → scripts/bootstrap_auditlens.py → src/product/bootstrap.py)
# only does credential collection, Kafka connectivity validation, and config
# file rendering. It does NOT run the production app — that lives in Docker
# (see requirements.txt, used by the forwarder/api/frontend images).
#
# Keep this list as small as possible: every package here has to install
# successfully on whatever Python version the operator's host happens to
# ship with. Heavier deps (pandas, numpy, pyarrow, fastapi, SQLAlchemy,
# pydantic-core, ...) used to live here too, which made first-time setup
# fail on Python 3.14 because those wheels weren't published yet. They
# don't belong on the host at all.
#
# Floor pins (>=) for native-extension packages so pip can resolve a wheel
# for the current Python without forcing a sdist build that would need a
# C toolchain on the host.
#
# Direct imports made by scripts/bootstrap_auditlens.py:
#   • src.product.bootstrap     → PyYAML, confluent_kafka
#
# Transitive imports loaded eagerly by src/product/__init__.py before the
# lazy bootstrap getattr fires:
#   • src.product.auth           → orjson
#   • src.product.persistence    → orjson
#
# If you add an import to any module that __init__.py loads at top level,
# update this file too. The setup wizard runs `python -c "import
# scripts.bootstrap_auditlens"` semantics; a fresh ModuleNotFoundError on
# any first-time install means a transitive dep is missing here.
confluent-kafka>=2.13.2
orjson>=3.11.9
PyYAML>=6.0.2
