# Brainstorming

Transform rough ideas into fully-formed designs through structured exploration.

## When to Use This Skill

- User says "help me think through..."
- User has a vague idea needing refinement
- User asks "what are my options for..."
- User needs to explore alternatives
- User says "brainstorm ideas for..."

## Process

### Phase 1: Clarify the Goal
Ask clarifying questions:
- What problem are you solving?
- Who is this for?
- What constraints exist?
- What does success look like?

### Phase 2: Divergent Thinking
Generate multiple alternatives without judgment:
- Aim for 5-10 options minimum
- Include wild/unconventional ideas
- Consider opposite approaches
- Draw from different domains

### Phase 3: Evaluate Options
For each option assess:
- Feasibility (1-5)
- Impact (1-5)
- Effort required
- Risks/unknowns

### Phase 4: Refine Top Choices
Deep dive on 2-3 best options:
- Detail implementation approach
- Identify dependencies
- Surface potential issues
- Create action plan

### Phase 5: Decision Support
Present recommendation with:
- Clear rationale
- Trade-offs acknowledged
- Next steps defined

## Techniques

### Six Thinking Hats
- White: Facts and data
- Red: Emotions and intuition
- Black: Risks and problems
- Yellow: Benefits and optimism
- Green: Creativity and alternatives
- Blue: Process and organization

### SCAMPER Method
- Substitute: What can be replaced?
- Combine: What can merge?
- Adapt: What can be modified?
- Modify: What can change scale?
- Put to other use: New applications?
- Eliminate: What's unnecessary?
- Reverse: What if opposite?

### Constraint Removal
- What if budget was unlimited?
- What if time wasn't a factor?
- What if you had any skill?
- What if there were no rules?

## Output Format

```markdown
## Brainstorm: [Topic]

### Problem Statement
[Clear definition of what we're solving]

### Options Explored

#### Option 1: [Name]
- Description: ...
- Pros: ...
- Cons: ...
- Feasibility: X/5

[Repeat for each option]

### Recommendation
[Top choice with rationale]

### Next Steps
1. [First action]
2. [Second action]
3. [Third action]
```

## Example

User: "Help me think through how to improve our onboarding"

Response:
1. Ask: Who is onboarding? New users? New employees?
2. Ask: What's the current flow? What's broken?
3. Generate 8 alternative approaches
4. Score each on feasibility/impact
5. Deep dive top 3
6. Recommend with action plan
