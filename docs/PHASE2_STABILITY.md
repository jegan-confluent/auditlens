# Phase 2 тАФ Stability & Data Integrity

Phase 2 follows commit `bebb70e` (Phase 1 hardening). All seven items from the
brief were implemented. The 468-test baseline grew to 481 passed (13 new tests)
with the same 5 skipped tests preserved. No frontend changes; the
`audit_forwarder.py` monolith was not restructured.

## Files changed

### Modified
- `backend/requirements.txt` тАФ added `alembic==1.18.4` and `slowapi==0.1.9`.
- `backend/app/db/database.py` тАФ pool tuning for Postgres (`pool_size=5`, `max_overflow=10`, `pool_timeout=30`, `pool_recycle=1800`), `statement_timeout=30s` connect_args, SQLite `PRAGMA foreign_keys=ON` listener; explanatory comment that Alembic is now the canonical schema-change mechanism on Postgres while `_ensure_audit_event_columns()` stays on the SQLite demo path.
- `backend/app/db/models.py` тАФ `AuditEventTriage.event_fingerprint` now declares a `ForeignKey("audit_events.event_fingerprint", ondelete="CASCADE")` so a fresh DB ships the cascade FK in the table definition.
- `backend/app/services/event_service.py` тАФ keyset cursor encode/decode helpers, `cursor` parameter on `list_events_result()`, `next_cursor` field on `EventListResult`, defensive pre-delete of triage rows in `cleanup_retention()` so legacy SQLite databases without the cascade FK do not leak orphans.
- `backend/app/services/filter_options_service.py` тАФ `LIMIT 500` on each distinct, ordered by frequency; module-level `cachetools.TTLCache(ttl=60s)` keyed by engine identity; `clear_filter_options_cache()` exposed for tests.
- `backend/app/services/system_service.py` тАФ `_ForwarderHealthCache` (5-second TTL, 0.5-second tight HTTP timeout); `get_forwarder_status()` now returns the cached snapshot, so `/ready`/`/pipeline/ready` are sub-millisecond on a cache hit even when the forwarder is slow.
- `backend/app/main.py` тАФ slowapi wired in: `default_limits=["200/minute"]`, custom middleware that exempts `/live`, `/ready`, `/pipeline/ready`, `/ingestion/ready`, `/health`, 429 handler installed.
- `backend/app/api/routes/events.py` тАФ `@limiter.limit("20/minute")` on `GET /events` and `GET /events/{id}`; `cursor` query parameter wired into `EventListResponse`.
- `backend/app/schemas/response.py` тАФ `EventListResponse.next_cursor: str | None = None`.
- `backend/tests/test_api.py` тАФ fixture now resets the forwarder-health cache, the filter-options cache, and the slowapi limiter; new tests added (see below).
- `Makefile` тАФ added `make migrate` target (`cd backend && alembic upgrade head`).

### Created
- `backend/alembic.ini` тАФ generated by `alembic init alembic`, kept as default.
- `backend/alembic/env.py` тАФ points Alembic at `backend.app.db.models.Base.metadata`, resolves `DATABASE_URL` from settings/env, runs SQLite migrations in batch mode.
- `backend/alembic/script.py.mako` тАФ default revision template (unmodified).
- `backend/alembic/README` тАФ default Alembic README (unmodified).
- `backend/alembic/versions/0001_baseline.py` тАФ codifies the current schema via `Base.metadata.create_all(checkfirst=True)`. Idempotent on existing DBs.
- `backend/alembic/versions/0002_ensure_decision_columns.py` тАФ additive `ALTER TABLE` patches mirroring `_ensure_audit_event_columns()`/`DatabaseWriter._ensure_columns()` for legacy Postgres deployments.
- `backend/alembic/versions/0003_triage_cascade_fk.py` тАФ adds `ON DELETE CASCADE` FK between `audit_event_triage` and `audit_events` (Postgres only; SQLite path uses the model FK + application pre-delete).
- `backend/app/core/limiter.py` тАФ shared slowapi `Limiter` instance to break the `main.py тЖФ events.py` import cycle.
- `backend/tests/test_migrations.py` тАФ Alembic baseline smoke tests against an in-memory-style temp SQLite DB.
- `backend/tests/test_db_engine.py` тАФ verifies Postgres engine sets `statement_timeout` + pool sizes; SQLite engine does not; `PRAGMA foreign_keys=ON` is wired.
- `docs/DATABASE.md` тАФ schema-change policy: all Postgres schema changes must go through Alembic; SQLite demo path keeps `_ensure_columns`.
- `docs/PHASE2_STABILITY.md` тАФ this file.

## Issues fixed

| Issue (severity from `AUDIT_REPORT.md`) | What changed and why |
|---|---|
| **#22 Medium тАФ No formal migration system** | Introduced Alembic with three baseline revisions. `_ensure_audit_event_columns()` retained for SQLite demo only; Postgres production now uses `make migrate`. Schema-change policy documented in `docs/DATABASE.md`. |
| **#12 High тАФ No statement timeout on the API DB session** | Postgres engines now ship `connect_args={"options": "-c statement_timeout=30000"}` plus an explicit pool (size 5, overflow 10, timeout 30 s, recycle 30 min). SQLite is unchanged. |
| **#18 Medium тАФ Retention cleanup leaves orphan triage rows** | Added `ON DELETE CASCADE` FK in `AuditEventTriage.event_fingerprint` (model + Alembic revision 0003) plus an explicit pre-delete inside `cleanup_retention()` as a fallback for legacy SQLite installs whose existing tables don't carry the FK. |
| **#20 Medium тАФ `_distinct()` filter options unbounded** | `LIMIT 500` per column with `COUNT(*) DESC` ordering, plus a 60-second `cachetools.TTLCache` keyed by engine identity. Reduces DB load on dashboard repaints. |
| **#19 Medium тАФ Synchronous `httpx` call in readiness probe** | Cached the forwarder `/health` snapshot (5-second TTL, 0.5-second tight timeout). Subsequent `/ready` and `/pipeline/ready` hits within the TTL skip the network entirely; the new test confirms <100 ms response even when the upstream is sleeping for 2 s. |
| **#21 Medium тАФ No rate limiting** | `slowapi` limiter wired into FastAPI: 200/min default per IP, 20/min on `GET /events` and `GET /events/{id}`. `/live`, `/ready`, `/pipeline/ready`, `/ingestion/ready`, `/health` are exempt via a path-aware middleware wrapper. |
| **#? High тАФ `OFFSET` pagination breaks beyond ~100k rows** *(scalability section of `AUDIT_REPORT.md`)* | Implemented keyset pagination on `GET /events`. New `cursor` query param + `next_cursor` field on `EventListResponse`; cursor encodes `(timestamp, id)` as URL-safe base64 JSON. Backwards-compatible: no cursor = first page (existing `OFFSET` behaviour preserved). Derived-filter mode still uses offset because its scan window has different semantics. |

| `backend/tests/test_api.py::test_keyset_pagination_rejects_malformed_cursor` | An invalid cursor returns HTTP 400 (translated from a `ValueError` in the service). |

## Remaining risks

Issues from `AUDIT_REPORT.md` that Phase 1 and Phase 2 did not address. Each
line gives the audit-report finding number, severity, and a one-line reason
for deferral.

- **#1 Critical тАФ Live Confluent API keys in working-tree `.env`.** Requires operator action in Confluent Cloud + redeploys; not a code fix.
- **#3 Critical тАФ `raw_payload_json` exposure.** Phase 1 redacted it for non-admin callers; further hardening (column-level encryption / TDE) is a deployment change, deferred.
- **#4 Critical тАФ Offset commit / DB write ordering in the forwarder.** Phase 1 stopped popping the buffer until the write succeeded; deeper "kill DB mid-flush" integration tests need a real Postgres in CI and are deferred.
- **#6 High тАФ `Dockerfile.alpine` placeholder digest pin.** Container hardening pass; out of scope for an API/runtime phase.
- **#7 High тАФ Streamlit dashboards have no authentication.** Product-cleanup / UI auth, not an API fix.
- **#8 High тАФ `deploy/docker/docker-compose.yml` ships `admin/admin` Grafana default.** Lives in deployment config; needs operator-managed secret.
- **#9 High тАФ ALB HTTPS listener commented out.** Infrastructure change requiring an ACM certificate.
- **#13 High тАФ Unmasked delivery-error logging in the forwarder.** Logging-hygiene pass on `audit_forwarder.py`; deferred to a forwarder-focused phase.
- **#15 High тАФ Outdated dependencies.** Wider dependency upgrade pass (FastAPI, lxml, pydantic, httpx, frontend Next.js audit). Deferred.
- **#16 High тАФ Frontend race condition on filter changes.** Frontend was explicitly out of scope for Phase 2.
- **#17 Medium тАФ Lazy `_triage()` query is N+1.** Bulk callers already use `get_triage_snapshots()`; eliminating the lazy property requires touching every caller and is deferred.
- **#23 Medium тАФ schema-watcher rewrites committed source code.** Architectural change to schema-watcher; deferred.
- **#24 Medium тАФ K8s secrets unencrypted; no NetworkPolicy.** Infrastructure work.
- **#25 Medium тАФ ECS egress `0.0.0.0/0`.** Terraform change; deferred.
- **#28 Medium тАФ Rate-tracker `_cleanup()` is `O(n)` per add.** Forwarder hot-path optimisation; deferred.
- **#29 Medium тАФ Regex compiled per event in CRN parser.** Forwarder change; deferred.
- **#30 Medium тАФ Clean dashboard files untracked in git.** Product-cleanup item.
- **#31 Medium тАФ Two parallel UIs maintained.** Strategy decision (per `DASHBOARD_GAP_ANALYSIS.md` Tier 1/2 plan); deferred.
- **#32 Medium тАФ No CSP / HSTS / X-Frame-Options.** Phase 1 added `X-Content-Type-Options`, `X-Frame-Options`, `Referrer-Policy`. CSP and HSTS need product-aware policy work and are deferred.
- **Items #33тАУ#50 тАФ Low.** Various cleanup tasks (gitignore patterns, `Makefile` shell-sourcing, `test.sh` cleanup, Dockerfile consolidation, etc.). Deferred to a dedicated cleanup pass.
