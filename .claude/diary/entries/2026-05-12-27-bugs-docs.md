## Key Decisions

### Auth-disabled dev-mode bypass approach
- **Decision**: Auth-disabled path emits `Role.VIEWER` (not ADMIN), but all three guard functions (`require_admin`, `_require_responder`, `require_admin` in admin.py) add `if not config.enabled: return` before any role check
- **Why**: BUG-1 fix (VIEWER not ADMIN) alone broke existing tests that used admin/responder endpoints in dev mode. The correct model is: dev mode bypasses the *guard* before role inspection, not by granting elevated roles
- **Alternative rejected**: Keeping ADMIN in auth-disabled path — wrong security model, widens blast radius if auth config leaks

### Auth config cache with env-var key invalidation (BUG-28)
- **Decision**: Cache tuple `(timestamp, config, enabled_flag_value)` — a change to `API_AUTH_ENABLED` invalidates the cache without waiting for TTL
- **Why**: Tests use monkeypatch.setenv; the old monotonic-only cache persisted stale config between tests within the same process

### HTML: RAG + collapsible cards for gap analysis
- **Decision**: JS toggle via `classList.toggle('open')`, open Correctness and Performance (RED) cards by default
- **Why**: Staff engineer audience skims for red items first; collapsible avoids overwhelming a long doc

### SVG architecture diagram inline (no PNG/external)
- **Decision**: Embedded `<svg>` inside `.arch-wrap` container, fully self-contained
- **Why**: User specified "no frameworks/CDN", fully self-contained HTML — external image would break offline viewing

## Challenges & Solutions

### BUG-1 cascading breakage
- **Problem**: Changing ADMIN→VIEWER for auth-disabled immediately broke two existing tests: `test_event_triage_lifecycle_basic` (needs responder) and `test_admin_retention_cleanup_allows_dev_mode` (needs admin)
- **Solution**: Add `if not config.enabled: return` early-return bypass in all three guard functions before any role check. Auth-disabled mode bypasses the guard entirely — the role on the actor object is irrelevant
- **What didn't work initially**: Just changing the role without adding the bypass to guards

### BUG-28 test cache isolation
- **Problem**: Module-level `_auth_config_cache` persisted between test runs when `monkeypatch.setenv` changed `API_AUTH_ENABLED`. Tests that expected auth-disabled and auth-enabled behaviour interfered with each other
- **Solution**: Include `enabled_flag = _os.getenv("API_AUTH_ENABLED", "false")` as the third element of the cache tuple. Any monkeypatched env var change immediately invalidates the cache

### BUG-002 non-atomic dedup
- **Problem**: `_is_duplicate()` + `_record_dedup()` acquired the lock twice. Two concurrent threads could both pass the duplicate check before either recorded the entry
- **Solution**: Merged into single `_check_and_record_dedup()` that holds `_dedup_lock` across both check and record operations

### Write tool requires prior Read even for new files
- **Problem**: `Write` tool returned "File has not been read yet" error for files that exist on disk even when I intended to fully overwrite them
- **Solution**: Always do a minimal `Read` (limit=3) before calling `Write` to satisfy the file-state requirement

## Patterns Noticed

### "Read before fix" was enforced throughout bug fixes
Every fix included reading the actual file content before editing, often revealing that the code differed from the summary's mental model (e.g., BUG-29 was already partially commented, BUG-13 had a slightly different loop structure).

### Single-flight pattern used twice
`_fetching: bool` flag (ForwarderHealthCache) and `_fetching: set[int]` (DbLatestEventCache) both implement the same thundering-herd prevention. Pattern: set flag before async work, clear in `finally`, return stale data if already fetching.

### Dialect guard on every Postgres-specific query
`if db.get_bind().dialect.name == "postgresql":` before `SET LOCAL statement_timeout` and before Postgres-specific SQL constructs. Without this, SQLite-based test fixtures blow up. Consistent across event_service, noise_service, filter_options_service, backfill_service.

### Commit message format: `fix: <short description> — BUG-N`
User's preferred commit message convention includes the bug ID at the end. One logical fix = one commit.

## Code Patterns Worth Remembering

### Atomic dedup check-and-record
```python
def _check_and_record_dedup(self, fingerprint: str, dest: str) -> bool:
    with self._dedup_lock:
        key = (fingerprint, dest)
        last = self._dedup.get(key)
        if last and (time.time() - last) < DEDUP_WINDOW_SECONDS:
            return True
        self._dedup[key] = time.time()
        self._call_count += 1
        if self._call_count >= DEDUP_PRUNE_EVERY_N_CALLS:
            self._prune_dedup_locked()
            self._call_count = 0
        return False
```

### Auth-disabled bypass in guard functions
```python
def _require_responder(...):
    config = _get_auth_config()
    if not config.enabled:
        return  # dev mode — bypass before any role check
    # ...rest of auth logic
```

### Existence cache with asymmetric TTL
```python
# True = table exists, cache forever
# False = table absent, re-probe after 30 s (catches post-migration window)
_existence_cache: dict[int, tuple[bool, float | None]] = {}
_EXISTENCE_FALSE_TTL = 30.0
# Store True: (True, None) — Store False: (False, time.monotonic() + 30)
```

### Backfill cursor — commit before advance
```python
batch_last_id = None
for event in batch:
    batch_last_id = event.id
    # process...
db.commit()  # commit first
if batch_last_id:
    last_id = batch_last_id  # then advance cursor
```

## Potential CLAUDE.md Rules

- When changing an auth guard's default role, also audit every guard function that short-circuits for dev mode — the bypass must be before role inspection, not a role upgrade
- Dedup locks must be held across both check AND record operations — never two separate acquisitions
- Cache invalidation should key on relevant env vars (not just monotonic time) when tests use monkeypatch.setenv to change auth config
- Write tool requires a prior Read even for files being fully overwritten — always do `Read(limit=3)` before `Write` on existing files
- For gap analysis HTML: open the RED-rated card by default so the blocker is immediately visible
- HTML documentation: inline SVG for architecture diagrams, dark mode (#0d1117 bg), no CDN/frameworks, no external images
- Behavioral assertion changes (test expected value corrected because old value was wrong) must be called out explicitly in commit message
