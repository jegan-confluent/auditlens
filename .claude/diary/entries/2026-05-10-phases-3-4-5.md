  Existing 248-line Makefile already defines `test`, `clean`, `logs`
  with semantics that conflict with the prompt's spec (`test` uses
  `python -m pytest` without unset-creds wrapper; `clean` does
  `find . -name '*.pyc' -delete`, not `docker compose down -v`). User
  chose to add only `setup/start/stop/restart/status` and leave
  existing targets alone. `make test` was broken pre-Phase-3 (no
  `python` binary in PATH); flagged but not changed.

- **Phase 4, decision_snapshot overlay for signal_type vocabulary
  bridge.** Forwarder's `enriched_event['signal_type']` is the **legacy**
  `auth_failure / authz_denial / None` from
  `src/classification/criticality.py`. The new `notifications.yml`
  filters on `action_required / attention / informational / noise`
  from `src/product/event_signals.py`. Solved by overlaying
  `decision_snapshot()` (which calls `classify_signal()` and returns

  Bind-mounting an optional file is awkward in compose v2 (missing

  Docker integration either add a bind mount themselves or place
  `notifications.yml` in `./secrets/` and set
  `NOTIFICATIONS_CONFIG=/run/secrets/notifications.yml`. Documented
  in `.env.example`.

- **Phase 5, actor_mappings.yml as highest-priority source, not as
  another `manual` map.** Existing `_identity_map()` reads the
  `IAM_MAPPING_FILE` JSON (default `data/iam_mapping.json`). Per spec,
  the new YAML mapping wins over everything — even that JSON file. So
  `_lookup_actor_mapping_override` runs **before** the existing
  `manual` source, with `actor_source='manual_mapping'` (distinct from

## Challenges & Solutions

- **Bootstrap entrypoint discoverable but not runnable.** The shim at
  `scripts/bootstrap_auditlens.py` had no `__main__` block. Running
  `python scripts/bootstrap_auditlens.py` would import and exit
  without dispatching. Plus `from scripts.deprecated...` failed
  because Python adds the script's directory to `sys.path[0]`, not
  the repo root. Solved by (a) adding 2-line dispatch to the shim,
  (b) setting `PYTHONPATH=$SCRIPT_DIR` in `./setup` before exec.

- **Phase 5 Step 5 false negative.** My validation script passed

  `_normalize_raw_actor` prefers `subject` over `actor`, so `raw`
  became 'Some Other Name' (no mapping match) instead of

- **Phase 5 backfill statement_timeout firing on production-sized

  exceeds 5s with no index. Status endpoint correctly reports the
  error. Flagged operator workaround (`CREATE INDEX CONCURRENTLY ...
  WHERE actor_display_name IN ('Unknown user', ...)` partial index)

## Patterns Worth Remembering

- **Hot-reload via mtime + lock.** Both `AuditLensNotifier._maybe_reload`
  and `ActorMappingFile._reload_if_changed` follow the same shape:
  `os.path.getmtime` → compare to last-known → re-read under lock →
  swap dict atomically. Cheap on every call; avoids restarts when
  config changes.

- **Single-flight admin job + status endpoint.** `backfill_service.py`
  module-level `threading.Lock` + state dict. POST acquires lock,
  flips status to "running", spawns daemon thread that creates its
  own session via `sessionmaker(bind=engine, ...)`, captures
  exceptions into the state dict. GET reads state under the same
  lock. Re-POSTs while running return the in-flight state instead
  of starting a duplicate.

- **Daemon-thread budget for off-thread webhook dispatch.** The
  `_send_with_retry` loop computes `deadline = time.monotonic() +
  THREAD_TIMEOUT_SECONDS` and aborts retries when the deadline
  passes. Net effect: each destination gets a fixed time budget
  (30s) regardless of retry count or HTTP timeout interactions.

- **Postgres-only `SET LOCAL statement_timeout`.** `if
  db.get_bind().dialect.name == "postgresql": db.execute(text("SET
  LOCAL statement_timeout = ...))"`. Without the dialect guard,
  SQLite-based tests blow up. Pattern already established in
  `event_service.py`, `noise_service.py`, `filter_options_service.py`.

## Potential CLAUDE.md Rules

- Multi-fix prompt protocol: run baseline pytest before any change, after each commit; one fix = one commit; never squash; never push without ask.
- When prompt assumes a file/flag that doesn't exist on disk, use AskUserQuestion before guessing — Rule 8 exists for this.
- For new YAML config files: gitignore the live file, commit `.example.yml`, hot-reload via mtime with `threading.Lock`-guarded swap.
- Admin async-job pattern: module-level `threading.Lock` + state dict + GET `/admin/.../status` endpoint; daemon thread builds its own SQLAlchemy session via `sessionmaker(bind=db.get_bind(), ...)`.
- Postgres-only `SET LOCAL statement_timeout`: always wrap in `if db.get_bind().dialect.name == "postgresql"` so SQLite tests don't blow up.
- Daemon-thread retry budgets: compute `deadline = time.monotonic() + N` once and check before each attempt — bound the worst-case duration regardless of retry count or HTTP timeout interactions.
- For docker-compose optional config files: prefer "notifier handles missing file gracefully" over "force a bind mount that breaks compose-up when source is absent."
